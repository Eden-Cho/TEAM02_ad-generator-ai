## 1. 데모 순서

### ① 배너 짚고 시작 (15초)

> "화면 위쪽 보시면 **사전 생성 샘플, 실제 API 호출 없음**이라고 떠 있습니다.
> 지금 이 데모는 미리 만들어 둔 결과를 보여주는 것이고, 여기서 돈 나가는 호출은 **하나도 없습니다.**"

👉 사이드바 배너도 한 번 가리킨다. **"이 문구는 모든 탭에 떠 있습니다."**

---

### ② 탭 1 — preserve / natural 비교 (2분) ★핵심

**Mac Mini 부터** 시작한다(가장 검증된 표본).

1. 사이드바에서 **Mac Mini** 선택
2. 좌우로 `preserve` / `natural` 상세페이지가 나란히 뜬다

> "왼쪽이 preserve, 오른쪽이 natural입니다. **preserve는 제품 픽셀이 원본 그대로**입니다.
> 배경만 새로 만들어서 합성했습니다. 오른쪽 natural은 제품까지 다시 그린 겁니다."

3. 판정 배지를 짚는다: preserve `usable_reference` / natural `review_required`

> "판정이 다릅니다. preserve는 쓸 수 있는 상태, natural은 **검토가 필요한 상태**입니다."

4. 아래로 스크롤해 **갤러리 컷**을 보여준다 (역할: hero / build / connectivity / lifestyle)

> "컷마다 역할이 있습니다. hero·build·connectivity는 생성했고,
> **lifestyle은 passthrough** — 업로드한 사진을 그대로 씁니다. **API 호출 0회**입니다.
> 그래서 4컷이어도 **이미지 생성은 3회**입니다. 이게 preview가 미리 계산해주는 값입니다."

5. **사과**로 전환 → preserve/natural 비교를 짧게
6. **썬스틱**으로 전환 → ⚠️ **여기서 반드시 짚는다**

> "썬스틱은 **preserve만 보입니다.** natural 세트는 오류가 있어서
> **기본 화면에서 아예 빼놨습니다.** 다음 탭에서 따로 보여드리겠습니다."

👉 이 순간이 "오류를 숨기지 않고, 그렇다고 기본 화면에 섞지도 않았다"를 보여주는 지점.

---

### ③ 탭 2 — 결함 개선 비교 (2분) ★발표의 하이라이트

탭을 열면 상단에 경고가 먼저 뜬다.

> "이 탭에는 **결함이 있는 샘플**이 들어 있습니다. 경고부터 띄웁니다."

**(a) 썬스틱 texture — 수정 전/후**

1. 왼쪽 `수정 전`: 고체 스틱 옆에 **크림 스와치**가 그려져 있다

> "제품은 **고체 스틱**인데 옆에 **크림을 짜서 바른 자국**이 그려졌습니다.
> 화장품에서 제형이 실제와 다르면 **표시 위반 리스크**입니다."

2. 오른쪽 `수정 후(v2)`: 스와치 없이 스틱만

> "제품 형태를 `solid_stick`으로 받으면 **크림·녹음·누출 표현을 코드에서 차단**하도록 고쳤습니다."

3. 🚫 **반드시 읽는다** — 화면의 빨간 경고를 그대로 짚으며:

> "그런데 여기 쓰여 있는 대로, **고친 건 이 한 컷입니다.
> 상세페이지 전체를 다시 만든 게 아닙니다.** 전체 재생성은 유료라서 팀과 합의 후에 하겠습니다."

**(b) 사과 ingredient — 수정 전/후**

> "사과는 성분 컷에 **입력에 없던 아몬드·호두·민트**가 생겼습니다.
> 없는 재료가 보이면 성분 오인이죠. 지금은 **사용자가 입력한 근거에서만** 조립하도록 바꿨고,
> 근거가 없으면 아예 넣지 않습니다. 오른쪽은 제품만 남았습니다."

**(c) 오류 참고 세트**

1. 아래 `오류 참고 세트 (운영 사용 금지)` → 접힌 영역을 **펼쳐서** 보여준다

> "썬스틱 natural 세트 6장입니다. **운영 사용 금지**로 표시했고 기본 화면에선 빠집니다.
> 지우지 않은 이유는 **비교 자료로 남겨야** 같은 실수를 다시 안 하기 때문입니다."

> **패키지 전체를 거부**합니다. 발표 자료가 조용히 바뀌는 일이 없게요."

## 3. 돌발 상황 대응

| 상황 | 대응 |
|---|---|
| 화면에 "데모 샘플 경로가 설정되지 않았습니다" | `MODEL_V2_DEMO_ASSET_DIR`가 안 잡힌 것. export 후 Streamlit **재시작** |
| 화면에 "데모 샘플 패키지를 읽을 수 없습니다" | 폴더에 `manifest.json`이 있는지, ZIP을 통째로 풀었는지 확인. (내부 정보는 화면에 안 뜬다 — 의도된 동작) |
| 이미지가 안 뜬다 | 패키지 폴더 안 `full_results/`·`fix_single_cuts/` 하위 파일이 다 풀렸는지 확인 |
| Streamlit이 안 켜진다 | 포트 충돌 가능 → `streamlit run ... --server.port 8503` |
| 상세페이지가 너무 길어 안 보인다 | 브라우저 확대 **67~80%**, 또는 갤러리 컷 위주로 진행 |
| "실제로 생성해보라"는 요청 | ⚠️ **즉석에서 하지 않는다.** "유료 호출이라 기본 차단해뒀고, 팀 합의 후 별도로 돌리겠습니다"라고 답한다 |
| 노트북 네트워크가 끊김 | **문제 없음.** 이 데모는 네트워크를 쓰지 않는다 |

## 4. 하지 말 것

- ❌ 데모 중 **유료 생성 버튼이 있는 테스트 UI**(`model_v2_app.py`)를 열지 않는다.
  발표용은 **`model_v2_demo_app.py`** 하나다.
- ❌ 오류 세트를 "이것도 결과물"처럼 소개하지 않는다 — **반드시 "운영 금지"** 라고 말한다.
- ❌ 수정 컷을 "전체 개선 완료"로 말하지 않는다 — **"단일 컷"** 을 매번 붙인다.
- ❌ 검증 안 한 카테고리를 "잘 됩니다"로 일반화하지 않는다.
- ❌ 화면에 API 키·`.env`·내부 절대경로가 뜨는 창을 띄우지 않는다.
